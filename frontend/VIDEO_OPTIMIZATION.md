# Оптимизация видео для веб

## Текущая проблема
Видео на фоне вызывает мигание при скролле, особенно заметно на navbar.

## Рекомендуемые параметры видео для веб:

### Оптимальные характеристики:
- **Разрешение**: 1920x1080 (Full HD) - максимум
- **Codec**: H.264 (High Profile)
- **Битрейт**: 2-5 Mbps (не больше!)
- **FPS**: 24-30 fps (60 fps - слишком тяжело)
- **Формат**: MP4
- **Длительность**: 10-30 секунд (loop)

### Если текущее видео слишком тяжелое:

#### Вариант 1: Оптимизировать через FFmpeg
```bash
ffmpeg -i luxury-car-video.mp4 -vcodec h264 -acodec aac -b:v 3M -r 30 -vf scale=1920:1080 optimized-video.mp4
```

#### Вариант 2: Временно заменить на статичный фон
Чтобы проверить, действительно ли видео - причина, можно временно отключить его.

#### Вариант 3: Использовать изображение вместо видео
Если проблема не решается, можно использовать качественное статичное изображение с легким CSS-анимированным градиентом.

## Тест без видео

Чтобы проверить, является ли видео причиной мигания:

1. Открой `components/Hero/Hero.module.css`
2. Добавь к `.videoContainer`:
```css
opacity: 0;
/* или */
display: none;
```

Если мигание пропадет - проблема точно в видео.

---

**Следующие шаги:**
1. Проверь текущие изменения - мигание должно уменьшиться
2. Если проблема осталась, попробуй тест с отключенным видео
3. Если без видео все хорошо - нужно оптимизировать видеофайл
